# L0.2 Increment 1 (v2, against current main) — Env Hygiene + Shim-Token-Off-Argv Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stop a spawned agent from reading the worker's secrets out of its own environment, and stop the per-agent mailbox bearer token from riding the agent's argv, by giving each agent a default-drop allowlisted environment and writing its whole MCP-servers map (mailbox + memory) to a `0600` file in its worktree passed via `--mcp-config <path>` instead of inline JSON.

**Architecture:** `agent._agent_env` changes from `dict(os.environ)` to a default-drop allowlist. A new pure helper `worker/mcp_config.py::write_mcp_config(worktree, mcp_servers)` writes `<worktree>/.skep/mcp.json` (`0600`, `fchmod` before the token bytes land) and best-effort-excludes it from git. `AgentProcess` gains `mcp_config_path` and `env_passthrough`; `_argv` prefers `--mcp-config <path>` over the old inline-JSON form. `Supervisor` gains an injectable `mcp_config_writer` seam, writes the map it already builds to the file, passes the path, and threads `agent_env_passthrough`. **This is a re-plan of the abandoned `feat/skep-l0.2-per-agent-isolation` branch against a `main` that has since gained the L1.1 agent-memory feature (an N-server MCP map + `--allowedTools` grant) and Sessions (`--model`/`--resume`/`--add-dir`).** **Honest scope:** this scrubs the agent's *own* environ and takes the token off argv (`/proc/<pid>/cmdline` is world-readable, so this genuinely closes the different-UID/other-local-user vector). It does NOT stop a *same-UID* sibling reading the worker's `/proc/<worker-pid>/environ` **or** reading the `0600` token file (same owner) — that closes only with Increment 2's PID/mount namespaces. Increment 1 is dependency-free surface reduction, not full containment.

**Tech Stack:** Python (stdlib `os`/`json`/`subprocess`/`secrets`, `dataclasses`), `pytest` (+ `pytest-asyncio`, `asyncio_mode=auto`), git worktrees.

```python
_CORE_ENV_KEYS = (
    "PATH", "HOME", "USER", "LOGNAME", "TERM", "LANG", "TZ", "SHELL",
)
# Kept only if present. Proxy vars matter because dropping HTTPS_PROXY on a
# proxied host kills the API and dropping NO_PROXY=127.0.0.1 sends the loopback

    "SSL_CERT_FILE", "SSL_CERT_DIR", "NIX_SSL_CERT_FILE", "LOCALE_ARCHIVE",
    "HTTP_PROXY", "HTTPS_PROXY", "NO_PROXY",
    "http_proxy", "https_proxy", "no_proxy",
)


def _agent_env(config_dir: str | None, *,
               passthrough: tuple[str, ...] = ()) -> dict[str, str]:
    """Build a default-drop allowlisted environment for a spawned agent.

    Scrubs the worker's own environment -- critically the entire SKEP_*
    namespace (SKEP_SHARED_SECRET, SKEP_DB), CLAUDE_CODE_*/CLAUDECODE session
    markers, and ANTHROPIC_* -- so a compromised agent cannot read them from
    its OWN environ. This does NOT stop a same-UID sibling reading the worker's
    /proc/<pid>/environ; that cross-process pivot closes only with L0.2
    Increment 2's PID namespace. Surface reduction, not full containment.

    CLAUDE_CONFIG_DIR is set explicitly from config_dir and never inherited.
    """
    src = os.environ
    env: dict[str, str] = {}
    for key in _CORE_ENV_KEYS:
        if key in src:
            env[key] = src[key]
    for key, val in src.items():
        if key.startswith("LC_"):
            env[key] = val
    for key in _OPTIONAL_ENV_KEYS:
        if key in src:
            env[key] = src[key]
    for key in passthrough:
        if key in src:
            env[key] = src[key]
    if config_dir is not None:
        env["CLAUDE_CONFIG_DIR"] = config_dir
    return env
```

### Task 3: `write_mcp_config` pure helper (new module)

**Files:**
- Create: `src/skep/worker/mcp_config.py`
- Test: `tests/worker/test_mcp_config.py` (`tests/worker/` already exists with no `__init__.py` — add none)

**Interfaces:**
- Consumes: nothing.
- Produces: `write_mcp_config(worktree: Path, mcp_servers: dict[str, dict]) -> Path` — writes `<worktree>/.skep/mcp.json` (`0600`, dir `0700`) containing `{"mcpServers": mcp_servers}`, best-effort-appends `/.skep/` to the worktree's git `info/exclude`, returns the file path. Takes the **whole servers map** the supervisor already builds (mailbox + memory), so the mailbox bearer token inside it lands in the file, never on argv.

**Design note (linked worktree):** a linked worktree's `.git` is a *file* and `info/exclude` resolves to the shared common dir, so resolve it with `git rev-parse --git-path info/exclude`, never `worktree/".git"/"info"/"exclude"`. The exclude is therefore **shared** across that repo's worktrees — assert only idempotent *presence* of `/.skep/`.

**Design note (fchmod is load-bearing):** for a non-`MODE_NEW` head root the worktree is a *persistent repo* (main's workspace model), so `mcp.json` can pre-exist at looser perms; `O_TRUNC` keeps the old mode, so `os.fchmod(fd, 0o600)` **before** `json.dump` — not just the trailing `os.chmod` — is what guarantees the token bytes never touch disk group/other-readable.

```python
"""Write a spawned agent's MCP-servers map to a 0600 file in its worktree.

Keeps the per-agent mailbox bearer token (inside the map's mailbox entry) OFF
the agent's argv -- out of /proc/<pid>/cmdline and `ps`, which are
world-readable, so this closes the different-UID / other-local-user vector. A
same-UID sibling can still read this 0600 file (same owner) and the worker's
/proc/<worker-pid>/environ; that closes only with Increment 2's mount/PID
namespaces. Increment 1's win here is purely removing the token from the
command line.
"""

from __future__ import annotations

import json
import os
import subprocess
from pathlib import Path


def _resolve_exclude_path(worktree: Path) -> Path | None:
    """This worktree's git info/exclude, or None if not a repo.

    For a linked worktree, info/exclude lives in the shared common dir;
    `git rev-parse --git-path` returns the correct path (relative to -C).
    """
    try:
        out = subprocess.run(
            ["git", "-C", str(worktree), "rev-parse", "--git-path",
             "info/exclude"],
            capture_output=True, text=True, check=True,
        ).stdout.strip()
    except (subprocess.CalledProcessError, FileNotFoundError, OSError):
        return None
    p = Path(out)
    return p if p.is_absolute() else worktree / p


def _exclude_skep_dir(worktree: Path) -> None:
    """Best-effort: append `/.skep/` to the worktree's git exclude (idempotent).

    Stops an agent's `git add -A` from committing its own token file. A git
    hiccup must NEVER abort a spawn -- the 0600 file is the real guard.
    """
    exclude = _resolve_exclude_path(worktree)
    if exclude is None:
        return
    try:
        existing = exclude.read_text() if exclude.exists() else ""
        if "/.skep/" in existing.splitlines():
            return
        exclude.parent.mkdir(parents=True, exist_ok=True)
        with exclude.open("a") as fh:
            if existing and not existing.endswith("\n"):
                fh.write("\n")
            fh.write("/.skep/\n")
    except OSError:
        return


def write_mcp_config(worktree: Path, mcp_servers: dict[str, dict]) -> Path:
    """Write `{"mcpServers": mcp_servers}` to a 0600 file in the agent's
    worktree and return its path. The token (in the mailbox entry) rides this
    file, not argv."""
    skep_dir = worktree / ".skep"
    skep_dir.mkdir(parents=True, exist_ok=True)
    os.chmod(skep_dir, 0o700)
    path = skep_dir / "mcp.json"
    fd = os.open(str(path), os.O_WRONLY | os.O_CREAT | os.O_TRUNC, 0o600)
    with os.fdopen(fd, "w") as fh:
        # BEFORE writing: O_TRUNC keeps a pre-existing file's looser mode, so a
        # stale mcp.json would otherwise expose the token during the write.
        os.fchmod(fd, 0o600)
        json.dump({"mcpServers": mcp_servers}, fh)
    os.chmod(path, 0o600)  # exact final mode regardless of umask
    _exclude_skep_dir(worktree)
    return path
```
